# Установка и запуск проекта RhythmFall
Эта инструкция поможет вам установить Conda (Miniconda), создать виртуальное окружение для проекта, установить все зависимости и запустить проект **RhythmFall**.

## Клонирование репозитория
Перед началом работы необходимо **склонировать** репозиторий проекта в выбранную директорию:
```bash
git clone https://github.com/abletoburntheweb/RhythmFall.git
```

## Установка Conda (Miniconda)

**Conda** – это менеджер окружений и пакетов, который позволяет изолировать зависимости проекта, чтобы избежать конфликтов между библиотеками и версиями Python.

Мы устанавливаем её, потому что некоторые нужные пакеты не ставятся напрямую через `pip` – у них могут отсутствовать готовые колёса (wheels) под Windows или требуются дополнительные системные зависимости, которые `pip` не умеет подтягивать сам. Conda же устанавливает такие библиотеки из собственных репозиториев вместе со всеми нужными компонентами, поэтому всё работает "из коробки".

Для проекта RhythmFall рекомендуется использовать **Miniconda** — облегчённую версию Conda без предустановленных пакетов, дабы исключить лишние зависимости, которые могут конфликтовать с версиями пакетов в `requirements.txt`.

### Подготовка Miniconda

Скачивание дистрибутива Miniconda выполняется с официального сайта [Miniconda](https://docs.conda.io/projects/miniconda/en/latest/).

Для установки необходимо нажать «Download» под Miniconda Installers.

#### Тип установки (Select Installation Type)
- **Install for: Just Me** – *рекомендуется*. Устанавливает Miniconda в каталог пользователя и не требует прав администратора.
- **Install for: All Users** – устанавливает Miniconda в системный каталог `C:\ProgramData\Miniconda3` и требует прав администратора.

> **При выборе варианта All Users** после завершения установки необходимо вручную добавить путь к Miniconda в системную переменную среды `Path`, чтобы команды `conda` и `python` были доступны из терминала.
> Путь, который требуется добавить:
> `C:\ProgramData\miniconda3\Scripts`
> Добавление выполняется через окно «Изменение системных переменных среды» – Переменные среды – Path – Изменить – Создать – Вставить путь.
> После этого необходимо перезапустить терминал или IDE, чтобы новые параметры вступили в силу.

#### Папка назначения (Choose Install Location)
По умолчанию Miniconda устанавливается в каталог пользователя, например:
```
C:\Users\<имя_пользователя>\miniconda3
```
Изменение пути не требуется. *Рекомендуется оставлять путь по умолчанию*.

#### Дополнительные опции (Advanced Installation Options)
Во время установки Miniconda предлагается несколько дополнительных опций. Настройка влияет на доступность команд и интеграцию с системой:
- **Create shortcuts (supported packages only)** — создаёт ярлыки для некоторых пакетов.
- **Add installation to my PATH environment variable** — добавляет Miniconda в системный PATH.
- **Register Miniconda3 as my default Python 3.13** — регистрирует Miniconda как версию Python по умолчанию для системы.
- **Clear the package cache upon completion** — очищает кэш пакетов после установки.

Все параметры можно оставить по умолчанию. Установка будет выполнена корректно и без их активации.

После завершения установки необходимо закрыть терминал или IDE, чтобы новые системные пути вступили в силу. После повторного открытия терминала можно переходить к созданию виртуального окружения для проекта.

## Создание виртуального окружения для проекта

После обновления системных путей создаётся изолированное окружение для проекта **RhythmFall** с помощью команды:
```bash
conda create -n rhythmfall python=3.10.11
```

Даже если в системе уже установлен другой Python (3.12 и тд), это не помешает: Conda создаст окружение с указанной версией Python 3.10.11, полностью независимой от системного интерпретатора.

В процессе выполнения команды необходимо согласие с условиями использования (ToS), введя `accept`.

После этого выполняется подтверждение продолжения операции (`Proceed ([y]/n)?`), необходимо согласие для продолжения. (Ввод `y`)

После успешного выполнения будет создано новое виртуальное окружение `rhythmfall`, готовое для установки всех зависимостей проекта.

## Инициализация Conda

После создания виртуального окружения необходимо выполнить команду:
```bash
conda init
```

По завершении выводится сообщение:
> "For changes to take effect, close and re-open your current shell."

Это означает, что текущая сессия терминала не видит обновлённые системные пути и настройки Conda. Для вступления изменений в силу необходимо закрыть терминал или IDE и открыть его заново. После этого можно продолжить работу с Conda и создавать виртуальные окружения.

## Настройка и активация виртуального окружения

### Добавление окружения в IDE

После создания виртуального окружения необходимо добавить его в IDE как новый интерпретатор:

1.  Необходимо нажать на интерпретатор в правом нижнем углу (Pycharm) – `Add New Interpreter` – `Add Local Interpreter…`
2.  В окне создания необходимо выбрать `Select existing` (Environment), так как само окружение было создано ранее
    -   **Тип:** `Conda`
    -   **Path to conda:** IDE определяет автоматически, но при необходимости можно указать вручную:
        `C:\ProgramData\Miniconda3\condabin\conda.bat`
    -   **Environment:** `rhythmfall`

### Активация окружения в терминале

После добавления интерпретатора *рекомендуется* закрыть и открыть терминал или IDE, чтобы обновились пути.

Для активации окружения используется команда:
```bash
conda activate rhythmfall
```

После успешной активации в начале строки терминала появится префикс `(rhythmfall)`, указывающий на активное виртуальное окружение.

-   Было: `C:\..\RhythmFall>`
-   Стало: `(rhythmfall) C:\..\RhythmFall>`

## Установка зависимостей

В проекте содержится файл `requirements.txt`, в котором перечислены все необходимые библиотеки и их версии.

Этот файл обеспечивает повторяемость установки и совместимость окружения с проектом.

Все зависимости устанавливаются в активное виртуальное окружение с помощью команды:
```bash
pip install -r requirements.txt
```

После завершения установки окружение полностью готово к работе с проектом.

## FAQ

### Команда `conda` не распознаётся в терминале
```
'conda' is not recognized as an internal or external command,operable program or batch file.
```

-   **Возможная причина:** переменные среды не обновились после установки Miniconda.
-   **Решение:** перезапустить IDE или терминал.
    Если не помогло – вручную добавить путь в системную переменную среды `Path`:
    `C:\ProgramData\miniconda3\Scripts`

### После ввода `conda create -n rhythmfall python=3.10.11` появляется ошибка
```
CondaError: cannot find conda environment: rhythmfall
```
или
```
'conda' is not recognized...
```

-   **Возможная причина:** Miniconda не была инициализирована после установки.
-   **Решение:** выполнить команду
    ```bash
    conda init
    ```
    и перезапустить терминал, чтобы изменения вступили в силу.

### Ошибка “PackagesNotFoundError” при установке зависимостей.
```
PackagesNotFoundError: The following packages are not available from current channels: - <название_пакета>
```

-   **Возможная причина:** несовместимость версии Python или отсутствующий канал.
-   **Решение:** использовать стабильную версию Python (3.10.11) при создании окружения.
    Если пакет недоступен в стандартных источниках, можно добавить дополнительный канал с открытыми сборками:
    ```bash
    conda config --add channels conda-forge
    ```
    Команда добавляет репозиторий `conda-forge` – крупное сообщество, поддерживающее тысячи пакетов, которых может не быть в официальных каналах Anaconda. После добавления этого канала повторная установка зависимостей обычно проходит успешно.

### PyCharm не видит окружение Conda.
```
Cannot find Conda executable
```

-   **Возможная причина:** PyCharm не смог автоматически определить путь к Miniconda.
-   **Решение:** вручную указать путь к `conda.bat`:
    `C:\ProgramData\Miniconda3\condabin\conda.bat`

### После установки зависимостей проект не запускается.
```
ModuleNotFoundError: No module named '<название_модуля>'
```

-   **Возможная причина:** окружение не было активировано перед установкой пакетов.
-   **Решение:** активировать окружение вручную:
    ```bash
    conda activate rhythmfall
    ```
    и повторно установить зависимости:
    ```bash
    pip install -r requirements.txt
    ```

### Ошибка “EnvironmentNotFound”
```
EnvironmentNameNotFound: Could not find conda environment: rhythmfall
```

-   **Возможная причина:** окружение было создано под другим именем или не сохранено.
-   **Решение:** Проверить список доступных окружений:
    ```bash
    conda env list
    ```
    Если окружения нет – создать заново:
    ```bash
    conda create -n rhythmfall python=3.10.11
    ```

## Советы по устранению проблем

Если ошибки продолжают появляться, можно попробовать следующие шаги:

1.  **Удалить и пересоздать окружение**
    ```bash
    conda remove -n rhythmfall --all
    conda create -n rhythmfall python=3.10.11
    ```
    После этого повторно активировать окружение и установить зависимости:
    ```bash
    conda activate rhythmfall
    pip install -r requirements.txt
    ```

2.  **Очистить кэш пакетов Conda**
    Иногда помогает, если установка зависимостей зависает или выдаёт ошибки скачивания:
    ```bash
    conda clean --all
    ```

3.  **Проверить пути в PyCharm**
    Если IDE всё ещё не видит окружение — вручную указать путь к `conda.bat`:
    `C:\ProgramData\Miniconda3\condabin\conda.bat`

4.  **Перезапустить IDE и терминал**
    После любых изменений в `Path`, активации окружения или установки пакетов перезапуск *обязателен* – иначе среда может не обновиться.
