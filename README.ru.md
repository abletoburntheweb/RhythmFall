# RhythmFall

Преобразует вашу музыку в играбельные уровни ритм‑игры. Полностью локальная автоматическая генерация нот — без ручного маппинга.

> **Скачать игру:** готовые сборки (Windows) с 20 треками и примерами нот доступны в разделе [Releases](../../releases).  
> **Примечание:** Этот репозиторий содержит клиентскую часть (Godot). Репозиторий сервера (анализ аудио и генерация нот): https://github.com/abletoburntheweb/RhythmFallServer

Языки: [English](./README.md) | Русский

## Что это
RhythmFall — ритм‑игра на Godot, которая анализирует выбранный аудиофайл и заранее генерирует играбельную разметку нот. Локальный Python‑сервер на базе ML‑моделей (Essentia, madmom, Discogs400) выполняет аудио‑анализ и возвращает готовые данные в клиент.

## Как это работает
- Клиент (Godot) отправляет путь к аудиофайлу и параметры генерации на локальный сервер.
- Сервер определяет BPM, детектирует ударные события, применяет жанровые паттерны и формирует ноты.
- Клиент сохраняет результат в `user://` — уровень готов к игре.

## Ключевые возможности
- Автогенерация нот из аудио (без ручной разметки)
- Паттерны для ударных: режимы `basic` (быстро) и `enhanced` (плотнее, с жанровыми заполнениями)
- Учёт жанра: плотность и грув подстраиваются под стиль трека
- Опциональное разделение на стемы для точной детекции ударных
- Полностью локальный процесс: обмен данными только с `localhost`

## Быстрый старт
1. Запустите локальный сервер (инструкции в репозитории сервера).
2. Откройте проект в Godot и запустите сцену игры.
3. В меню генерации выберите: инструмент `drums`, режим (`basic`/`enhanced`), число дорожек (3–5) и трек.
4. Дождитесь завершения генерации и играйте.

## Примечания
- Готовые сборки игры (включая 20 треков и примеры сгенерированных нот) доступны в [Releases](../../releases) в виде `.zip`-архива с `.exe`.
- Исходный репозиторий не включает аудиофайлы; при сборке из исходников музыку необходимо добавить отдельно.
- Анализ и генерация выполняются на вашем устройстве; треки не передаются во внешнюю сеть.
- Качество генерации зависит от микса и жанра: режим `enhanced` даёт более детальные паттерны, но требует больше времени и ресурсов.
- Использование стемов (`use_stems`) повышает точность детекции ударных, но увеличивает время обработки — отключайте для быстрых тестов.
- Поддерживаются форматы: MP3, WAV, OGG. Нестандартные кодеки могут не обрабатываться.
- Для тонкой настройки пайплайна и параметров анализа обращайтесь к документации сервера.
